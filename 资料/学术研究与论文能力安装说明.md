## 推荐使用方式

### 1. 做选题和找论文

优先使用：

- `semanticScholar`
- `paperSearch`
- `academic-researcher`

适合的问题：

- “帮我找 2023-2026 年关于多模态检索增强生成的高被引论文”
- “总结这 20 篇论文的方法分类和趋势”
- “找与某篇论文最相关的后续工作”

### 2. 写 related work / 文献综述

优先使用：

- `literature-review`
- `academic-researcher`
- `semanticScholar`

适合的问题：

- “帮我把这些论文整理成三条技术路线”
- “找出目前方法的共同局限和研究空白”
- “根据这些论文生成 related work 提纲”

### 3. 管理引用和 BibTeX

优先使用：

- `bibtex`
- `zotero` MCP
- `zotero` skill

适合的问题：

- “帮我生成这篇论文的 BibTeX”
- “检查我的参考文献格式”
- “从 Zotero 里找这篇论文和附件内容”

## 注意事项

### 1. 需要重启 Codex

skills 和新增 MCP 写入配置后，建议重启 Codex，再开始调用这些能力。

### 2. Zotero 使用前提

如果你要用 `zotero` MCP：

- 本机需要安装并打开 Zotero
- 默认按本地 API 模式配置
- 如果你之后想改成 Web API 模式，需要额外提供：
  - `ZOTERO_LIBRARY_ID`
  - `ZOTERO_API_KEY`

### 3. Semantic Scholar API Key 是可选项

`semanticScholar` 不配 key 也能跑，但速率限制会更紧。

如果你后面想提速，可以自己申请 `Semantic Scholar API key`，再把它加到 `config.toml` 的 `semanticScholar.env` 里。

### 4. 本次 Python 安装带来了一点依赖冲突提示

安装 MCP 包时，`pip` 提示了两个已有环境冲突：

- `streamlit 1.37.1` 期望 `packaging < 25`
- `streamlit 1.37.1` 期望 `rich < 14`

这次安装没有阻断 MCP 包本身，但如果你本机另有依赖 `streamlit` 的项目，后续可能需要单独做虚拟环境隔离。

## 这批能力最适合做什么

如果你主要是写论文，推荐你平时这样组合：

- 选题调研：`semanticScholar + paperSearch + academic-researcher`
- 文献综述：`literature-review + semanticScholar`
- 引用管理：`bibtex + zotero`
- 成稿打磨：继续用你原来已有的 `ml-paper-writing + paper-self-review + academic-paper-reviewer`
