# Scouter 论文实验详细总结

## 一、论文核心目标

Scouter 的目标是：

> 根据 **control cell 的表达** 和 **被扰动基因的 LLM embedding**，预测该基因扰动后的转录响应。

它是一个**专门为 unseen perturbation prediction 设计的轻量模型**，而不是通用 foundation model。  
论文说明，Scouter 输入 control cell expression 和 perturbed gene 的 LLM embedding，输出预测的 perturbed expression，并且针对的是训练集中未见过的 perturbations。

---

## 二、实验总览

Scouter 主要做了以下几类实验：

1. 单基因扰动预测主实验  
2. 单基因案例可视化实验  
3. 双基因扰动预测实验  
4. 双基因案例可视化实验  
5. LLM embedding 截断实验  
6. 替换 GEARS / biolord embedding 的消融实验  
7. 与 foundation models 的对比实验  

---

## 三、单基因扰动预测主实验

### 1. 实验目的

验证：

> 如果训练时没见过某个基因的 perturbation，Scouter 能不能预测这个基因被 perturb 后的表达变化？

### 2. 数据集

使用 5 个 Perturb-seq 数据集：

- Dixit
- Adamson
- Norman
- Replogle K562
- Replogle RPE1

### 3. 对比方法

- GEARS
- biolord

### 4. 评估指标

沿用 GEARS / biolord 的标准：

- normalized MSE
- 1 − normalized PCC

并且都在 **top 20 DEGs** 上计算。

### 5. 结果

论文报告：

- Scouter 在五个数据集上都取得最低的 MSE 和 1−PCC
- 平均来看，误差约只有 biolord 和 GEARS 的一半左右

### 6. 说明了什么

说明 Scouter 对 unseen perturbations 的外推能力是成立的，  
并且在关键 DEGs 上比现有专用方法更强。

## 五、双基因扰动实验

### 1. 实验目的

验证：

> 对两个基因同时 perturb 的组合场景，Scouter 能否预测表达响应？

### 2. 数据集

使用 Norman 数据集，因为它包含双基因 perturbation。

### 3. 方法

Scouter 通过：

- 将两个基因的 ChatGPT embedding 相加
- 将和向量作为组合扰动输入

### 4. 难度分组

和 GEARS / biolord 一样，分成：

- seen2：两个基因都在训练中见过，但没同时出现
- seen1：只见过其中一个
- seen0：两个都没见过

### 5. 指标

- normalized MSE
- 1 − PCC

### 6. 结果

论文指出：

- Scouter 在 seen0 / seen1 / seen2 各组都优于 biolord 和 GEARS
- 而且在双基因场景中的优势更明显

### 7. 说明了什么

说明 Scouter 学到的不是简单的单基因模板，  
而是具备一定 gene-gene interaction 的组合外推能力。

- 相比 GO-graph-based 方法，它更不受知识图覆盖限制

## 七、LLM embedding 截断实验

### 1. 实验目的

验证：

> Scouter 的提升是否真的来自基因文本描述中的语义信息？

### 2. 做法

把基因的 NCBI textual summary 只保留前 K% 的词，再重新生成 embedding，观察性能变化。

### 3. 说明了什么

如果文本被截断后性能下降，就说明：

- 模型确实在利用基因文本里的语义信息
- embedding 不是随机标签向量

---

## 八、替换 GEARS / biolord embedding 的消融实验

### 1. 实验目的

验证：

> Scouter 的优势到底来自 LLM embedding，还是来自更适合该 embedding 的网络结构？

### 2. 做法

把 GEARS / biolord 原本的 GO-based embedding，替换为 LLM-derived embedding。

### 3. 结果

论文指出：

- 替换后确实有改善
- 但改善幅度有限

### 4. 说明了什么

这表明 Scouter 的成功来自两部分共同作用：

- LLM gene embedding
- 更匹配这种高维 dense embedding 的 compressor–generator 结构

---

## 九、与 foundation models 的对比实验

### 1. 实验目的

验证：

> 在 perturbation prediction 任务上，Scouter 与 foundation models 相比表现如何？

### 2. 对比模型

- scGPT
- scELMo
- scFoundation

### 3. 数据集

- Adamson
- Norman

### 4. 结果

论文结论：

- 在单基因和双基因扰动场景中
- Scouter 都显著优于这些 foundation models

### 5. 额外优势

作者还强调：

- Scouter 不需要大规模预训练
- 可在单个 Perturb-seq 数据集上端到端训练
- 普通 GPU 甚至 CPU 即可训练
- 最大数据集下也能在 1 小时内完成训练

### 6. 说明了什么

说明在 perturbation extrapolation 这个特定任务上，  
一个为任务专门设计的轻量模型，可能比通用 foundation model 更合适。

---

## 十、一句话总结

Scouter 的实验链条非常清晰：

> 先用单基因 / 双基因 benchmark 证明它的 unseen perturbation prediction 很强，  
> 再用具体案例说明它在 GO coverage 不足时仍然能工作，  
> 再通过 embedding 截断和替换实验说明它的提升来自 **LLM embedding + 合适架构**，  
> 最后再用与 scGPT / scELMo / scFoundation 的对比证明：在 perturbation prediction 任务上，Scouter 这个专用轻量模型比 foundation model 更强。
