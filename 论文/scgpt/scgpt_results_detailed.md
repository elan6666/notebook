# scGPT 论文详细讲解（二）
## 对应部分：2 Results

> 这一份专门讲 Results。  
> 不只是“论文做了哪些实验”，而是要讲清楚：**每个实验在验证 scGPT 的哪一种能力**，以及这些结果对我们理解 foundation model 有什么意义。

---

# 0. Results 部分应该怎么读？

scGPT 的 Results 不是一串孤立 benchmark，而是一条逐步展开的论证链：

1. 先证明 **细胞表示是可用的**（cell type annotation）；
2. 再证明 **表示可以跨批次、跨数据域对齐**（batch integration / reference mapping）；
3. 再证明 **表示可以跨模态统一**（multi-omic integration）；
4. 再证明 **模型具有生成能力**（perturbation prediction / reverse perturbation）；
5. 最后证明 **模型还学到了基因层结构**（gene embeddings / attention-based interaction）。

所以这部分不能只看表格分数，而要看每一个实验在整个大论证里扮演什么角色。

---

# 1. 细胞类型注释（Cell type annotation）

## 1.1 这组实验在验证什么？

这个实验验证的是最基础的一点：

> 预训练后的 scGPT，能不能学到一个足够好的 **cell embedding**，使它在细胞类型分类任务上具备竞争力甚至优势？

如果连这个都做不好，那 foundation model 的故事很难成立。  
因为细胞类型注释几乎是单细胞表征学习最基本的试金石。

---

## 1.2 为什么作者先做这个任务？

因为 cell type annotation 是最容易理解的下游任务：

- 输入：单细胞表达；
- 输出：细胞类别；
- 指标：accuracy / macro F1 / 混淆矩阵等。

它是判断模型表示是否“有判别力”的最直接方式。  
如果 scGPT 的 embedding 能把不同细胞群清楚分开，说明模型确实学到了细胞状态结构。

---

## 1.3 从结果图应该怎么看？

论文在这一部分通常会展示：
- UMAP 可视化；
- 混淆矩阵；
- embedding 热图；
- 不同数据集上的分类指标对比。

你可以这样理解：

### （1）UMAP 可视化
如果预测标签和原始注释在 UMAP 上的空间结构接近，说明模型的表征与真实生物分群是一致的。  
这不是最终证据，但很直观。

### （2）混淆矩阵
它回答两个问题：
- 模型最容易混淆哪些细胞类型？
- 错误是随机的，还是发生在生物学上相近的细胞之间？

如果混淆主要发生在发育/功能相近群之间，通常说明模型不是完全瞎猜，而是在边界处犯错。

### （3）embedding 热图
这类图不是为了“好看”，而是说明：
- 预训练后的 embedding 是否已经把同类细胞压到相似区域；
- 不同细胞类型是否在高维空间形成可分结构。

---

## 1.4 这一结果真正说明什么？

如果 scGPT 在 pancreas、myeloid、MS 等数据集上表现较强，那最核心的结论不是“它分类准”，而是：

### 结论一：预训练帮助模型学到了通用细胞状态表示
也就是说，它不是从零开始硬记标签，而是能把预训练知识迁移过来。

### 结论二：foundation model 的表示在监督任务中是可利用的
很多时候，预训练模型在看起来很强，但真正微调后不一定有收益。  
scGPT 用这一组实验说明：它的预训练不是空的，确实对下游有帮助。

### 结论三：embedding 不只可视化漂亮，还能支撑任务头
这对后面 batch integration、perturbation 等任务非常重要。  
因为这些任务都默认 backbone 的 cell representation 是可信的。

---

## 1.5 需要保留的中立看法

虽然 cell type annotation 是很好的起点，但它也有边界：

1. 这个任务通常比较依赖训练 / 测试分布相近；
2. 高性能不代表模型能处理 novel cell type；
3. 分类强不等于生成强，也不等于机制理解强。

所以这一部分只能证明：
**scGPT 至少是一个强细胞表征模型。**

---

# 2. 参考映射 / 疾病映射（Reference mapping）

## 2.1 这部分在验证什么？

如果说 cell type annotation 验证“表征是否有判别力”，  
那 reference mapping 验证的是：

> 模型能不能把 query 数据稳健地映射到 reference 空间中，同时保留生物差异、减少技术差异？

这对于现实单细胞分析非常关键。  
因为很多实际场景不是重新从零做注释，而是：

- 已有一个健康 reference atlas；
- 来了一批新样本（疾病、不同平台、不同实验批次）；
- 希望把它们映射到已有参考空间上。

---

## 2.2 为什么这是 foundation model 的重要能力？

因为 foundation model 不应该只会“在固定训练集里分类”，  
它更应该具备一种“把新数据纳入已有知识空间”的能力。

reference mapping 实际上考验三件事：

1. **表示的稳定性**：新数据进来后，空间不会完全乱掉；
2. **跨域泛化**：不同来源数据能被放到统一嵌入空间；
3. **生物信号保留**：疾病相关变化不能在对齐中被抹平。

---

## 2.3 这个结果该如何理解？

作者在疾病数据上做 reference mapping，本质上是在问：

- 健康细胞和疾病细胞能否在共享空间中被合理定位？
- 疾病相关亚群是否能被识别出来？
- 模型能否在“对齐”与“保留差异”之间取得平衡？

这是比简单分类更难的问题。  
因为过强的对齐会把疾病特异信号也消掉；  
而对齐不够又会导致 query 完全漂在 reference 外面。

如果 scGPT 在这类任务上表现好，说明它的 embedding 空间不仅能分类，还能作为跨数据集的坐标系统使用。

---

# 3. 扰动响应预测（Perturbation prediction）

## 3.1 这是 Results 里最关键的一部分之一

因为这一部分第一次真正体现出 scGPT 的“生成性”。  
前面的注释和整合主要还是表示学习；  
而 perturbation prediction 需要模型去回答：

> 给定一个细胞初始状态和一个扰动条件，扰动后表达谱会变成什么样？

这不再是分类问题，而是条件生成 / 条件预测问题。

---

## 3.2 为什么这个任务难？

扰动预测的难点主要有四个：

### 难点一：细胞状态依赖
同一个 perturbation，在不同细胞状态中效应可能不同。

### 难点二：高维输出
输出不是一个类别，而是成千上万个基因的表达变化。

### 难点三：组合爆炸
基因扰动组合很多，不可能都见过。

### 难点四：真实信号与技术噪声混杂
单细胞表达本身就稀疏、噪声大，预测 perturbation effect 进一步放大了难度。

所以如果一个 foundation model 能在这里成功，说明它学到的不是浅层分群信息，而是更深层的条件响应结构。

---

## 3.3 scGPT 在这里到底展示了什么能力？

### 能力一：从 control state 推断 perturbed state
这说明模型不只是记忆某个 perturbation 的平均效应，而是把“细胞状态 + 条件”作为联合输入来建模。

### 能力二：迁移到未见组合或更难场景
如果在 unseen perturbation / combinatorial perturbation 上仍保持竞争力，说明模型内部存在可组合的响应表示。

### 能力三：支持 reverse perturbation
论文还讨论了 reverse perturbation，这个概念很有意思。  
它不是只问“给定 perturbation，结果怎样”，  
还会反过来问“从目标表达状态出发，什么 perturbation 可能把细胞推到那里”。

这说明 scGPT 的条件生成框架具备一定双向推理潜力。

---

## 3.4 这一结果为什么重要？

因为它把 scGPT 从“强表征模型”推进成“可干预建模模型”。

单细胞 foundation model 如果只能做 embedding，其实还停留在分析工具层面；  
而 perturbation prediction 把它带到更接近**机制建模与 in silico experiment** 的层面。

这也是为什么这部分对药物扰动研究尤其关键。  
它表明 foundation model 可以成为：
- 扰动效果预测器；
- 条件生成器；
- 潜在实验筛选器。

---

## 3.5 但要避免过度解读

即使结果很好，也要谨慎：

1. 预测对不代表机制完全正确；
2. 平均表达预测准，不代表所有细胞亚群都准；
3. 组合扰动泛化强，不代表真正 OOD 生物机制已经被理解；
4. 反向扰动更接近“候选生成”，不是直接因果识别。

因此，这部分的最佳理解是：

> scGPT 展示了 foundation model 在单细胞扰动建模上的潜力，但仍应被视为强预测工具，而不是完整机制模型。

---

# 4. 多批次整合（Multi-batch integration）

## 4.1 这部分在验证什么？

multi-batch integration 验证的是：

> scGPT 的表示空间能否去除技术批次差异，同时保持真实生物结构？

在单细胞领域，这个问题极其基础。  
因为不同实验批次、平台、实验室、处理流程都会引入明显 batch effect。

---

## 4.2 为什么这对 scGPT 很关键？

因为 scGPT 的输入来自大量异质数据。  
如果模型本身不能处理 batch variation，那么大规模预训练反而会被噪声污染。

换句话说，foundation model 的前提之一就是：
**它必须能在异构来源数据中抽取稳定共性，而不是把批次当成语义。**

---

## 4.3 结果应该怎么理解？

这类实验通常会同时看两类指标：

### 一类是 batch mixing 指标
看不同批次是否在嵌入空间中混合得更好。

### 一类是 biological conservation 指标
看同一细胞类型 / 生物状态是否仍然聚在一起，没有被过度校正打散。

最难的地方就在于两者要平衡。  
如果一个方法只追求 batch mixing，很容易把所有细胞都揉成一团；  
如果只保留生物差异，又可能批次根本没消掉。

如果 scGPT 在两类指标上都强，说明它学到的是“去技术噪声但保留细胞身份”的表示。

---

## 4.4 为什么这一部分对 foundation model 尤其有说服力？

因为 batch integration 实际上是在检验：
**模型是否真正具备跨数据域的一致语义空间。**

这比单数据集分类更接近 foundation model 的本质。  
一个 foundation model 不应该只在同分布数据里有效，而应该能够吸收多源数据并在统一空间中组织它们。

---

# 5. 多组学整合（Multi-omic integration）

## 5.1 这部分验证的是更高一级的“统一性”

如果说 batch integration 是跨数据来源，  
那么 multi-omic integration 是跨观测模态：

- RNA
- ATAC
- 蛋白
- 其它单细胞组学形式

这一步更难，因为不同模态不仅噪声不同，连观测对象和统计分布都不同。

---

## 5.2 为什么 scGPT 能做这个任务？

因为它的输入设计从一开始就不是只为 RNA 准备的。  
论文里 condition token / modality token 的思想非常关键：

> 模型把“当前看到的是什么模态”作为上下文的一部分，而不是强迫所有模态共享完全同质的输入。

这意味着模型有机会学习：
- 同一细胞在不同模态下的对应关系；
- 不同模态之间的共享生物语义；
- 模态特异信号与共享信号的区分。

---

## 5.3 结果真正说明了什么？

如果 scGPT 在 multi-omic integration 上表现强，可以得出两层结论：

### 第一层：token 化方案有扩展性
说明基因 / peak / feature 的统一表示不是纸上谈兵，至少在实践中能支撑跨模态学习。

### 第二层：预训练 backbone 具备跨模态迁移能力
这对 foundation model 很关键。  
因为真正的生物基础模型，不可能永远只停留在单一 RNA 模态。

---

## 5.4 这一部分的意义比“整合效果好”更大

它实际上在证明：  
**scGPT 的统一建模接口不是只适配一种数据类型，而是具备走向多组学 foundation model 的可能。**

也就是说，这部分实验回答的不是“这个任务赢没赢”，  
而是“scGPT 这条技术路线能不能扩展到更广泛的生物数据世界”。

---

# 6. 基因 token embedding 分析（Gene embedding analysis）

## 6.1 为什么论文要专门分析 gene embedding？

因为如果 scGPT 真是 foundation model，它不应该只产出 cell embedding。  
它还应该给出一个有意义的 gene representation space。

作者分析 gene embedding，本质上是在问：

> 模型内部的基因向量空间，是否反映了真实的生物学关系？

---

## 6.2 这个实验在验证什么？

这部分通常是看：
- 功能相近基因是否在 embedding 空间靠近；
- 某些 gene program 是否能在 embedding 图中聚成模块；
- embedding 结构是否对应已知生物通路或功能类别。

如果答案是肯定的，就说明模型不是单纯为了下游头临时优化，而是真正在内部构建了“基因知识几何”。

---

## 6.3 为什么这一步很重要？

因为这说明 scGPT 的知识载体是双层的：

- **细胞层**：细胞是什么状态；
- **基因层**：哪些基因在功能上、调控上彼此接近。

这对后续很多任务都很关键：
- gene program discovery
- pathway analysis
- perturbation effect decomposition
- gene network inference

也就是说，gene embedding 不是附赠品，而是 foundation model 能否支持更深层生物分析的核心资产。

---

# 7. 基于注意力的基因相互作用分析（Attention-based gene interaction analysis）

## 7.1 这部分是论文里“可解释性”最浓的一块

作者不仅要说模型能预测，还要说模型内部的 attention pattern 可能帮助发现基因间相互作用。

这里要非常精确地理解：

- 论文不是说 attention 就是真实 GRN；
- 论文是在说 attention / embedding 里可能蕴含可用的交互信号。

---

## 7.2 这部分到底在验证什么？

它想验证两个层面：

### 层面一：模型有没有学到 gene-gene dependency？
如果没有，那 attention 分析就没有意义。

### 层面二：这些 dependency 是否和已知生物知识有对应？
例如某些基因对在已知通路、功能模块、免疫相关系统中确实有关联。

如果模型抽出来的交互结构与已有知识吻合较多，就说明它的内部关系学习不是纯噪声。

---

## 7.3 为什么这对 foundation model 很关键？

因为 foundation model 的价值不该只停留在“做一个更强黑箱预测器”。  
更高的目标是：
- 从模型中抽出可供科学家分析的知识结构；
- 让模型成为 hypothesis generator；
- 帮助发现潜在调控程序。

这部分实验正是在为这个目标做第一步铺垫。

---

## 7.4 但这一块必须保留方法论谨慎

一定要记住下面几点：

1. 注意力权重不直接等于调控强度；
2. 高 attention 可能反映共表达、上下文依赖或模型需要，不一定是直接调控；
3. 模型内部结构的生物解释需要独立验证。

所以最稳妥的结论是：

> scGPT 的 attention 和 gene embedding 为基因关系分析提供了有价值的候选结构，但这些结构更适合作为生物假设生成的起点，而不是终点。

---

# 8. 如何把整个 Results 串成一句话？

Results 不是“做了很多任务”，而是在逐层证明：

- scGPT 学到的 **cell embedding** 能做判别；
- 这个 embedding 能跨批次、跨数据域、跨模态迁移；
- 模型还能做条件生成，预测扰动响应；
- 更进一步，模型内部还有可分析的 **gene-level knowledge**。

所以它最终支撑的是这样一个结论：

> scGPT 并不是一个单任务模型，而是一个在细胞层和基因层都沉淀了可迁移知识的预训练 backbone。

---

# 9. 你读 Results 时最该抓住的三个评价维度

## 维度一：统一性
是不是同一个 backbone 支撑了多个任务，而不是每个任务重新造模型。

## 维度二：迁移性
预训练知识是否真的帮助了微调和泛化，而不是只是参数初始化。

## 维度三：知识性
模型内部表示是否不仅有用，而且带有生物学结构。

---

# 10. 一句话总结 Results

> Results 部分真正证明的，不只是 scGPT 在若干 benchmark 上表现优秀，而是它在“细胞判别、跨域对齐、跨模态统一、扰动生成、基因关系抽取”这五个维度上都展示出了 foundation model 应有的统一性与迁移性。
