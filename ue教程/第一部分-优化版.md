# UE 客户端（系统向）零基础学习计划（24 周｜1h/天）
## 第 1/3 部分：Week 1–8（工程基础 + UE C++ 核心机制）

> 每天 60min：10min 回顾 + 25min 学习/阅读 + 25min 编码/提交  
> 交付习惯：每天至少 1 次 commit；每周形成 1 份可展示的“验收产物”

---

## 使用说明（优化版）
- **学习节奏**：每天 60 分钟，固定为 `10 分钟回顾 + 25 分钟学习 + 25 分钟动手`
- **执行原则**：每天只完成当天任务，不追求额外扩展；优先保证“能跑通、能记录、能复现”
- **提交规范**：每天至少 1 次 commit；每周至少形成 1 个可展示产物
- **Obsidian 建议目录**：
  - `01-notes/`：当天笔记
  - `02-demos/`：演示工程与截图
  - `03-reports/`：周报、复盘、性能/测试报告
  - `04-cheatsheets/`：宏、日志、命令、常见坑速查

## 第一部分的资源优先级
1. **主线**：Epic 官方文档 / Microsoft Learn / cppreference  
2. **辅线**：你那套 76 集 UE 入门视频  
3. **目标**：本部分结束后，你应具备“能看懂 UE C++ 工程骨架 + 能写最小系统模块”的能力，而不是只会蓝图搭 demo

## 与 76 集入门视频的对照关系
> 如果你想把视频一起学，建议这样对照，不必逐集硬刷：
- **Week 1**：P1–P10（安装、项目、关卡、Viewport）
- **Week 2**：P11–P20（Content Browser、Details、World Outliner、材质）
- **Week 3**：P21–P29（灯光、体积、基础场景、Blueprint 引入）
- **Week 4**：P30–P39（变量、函数、流程控制、Timeline、电梯/门/敌人）
- **Week 5**：P40–P49（GameMode、Pawn、Character、Input、Collision、Damage）
- **Week 6**：P51 起作为补充（UMG 预热），但本计划仍以官方文档为主
> 原因：视频适合建立 UE 使用感；官方文档更适合支撑“系统向岗位”的工程能力。

## 每周固定输出模板
- `weekN-summary.md`：本周总结
- `weekN-checklist.md`：验收清单
- `weekN-questions.md`：卡住的问题与解决方案
- `weekN-demo/`：最小演示工程或截图

## Week 1｜环境 + Git + 第一个 UE C++ 工程跑通
### 本周目标
- 跑通：创建工程 → 编译 → 运行 → 提交 Git
- 建立最小工程化：README、提交信息、日志输出

### Day 1（Git 安装与最小提交）
- 学：Git 安装、仓库、提交、日志
- 做：安装 Git；建空仓库；练 `init/add/commit/log`

### Day 2（分支/合并/冲突）
- 学：branch/merge；冲突本质（同一段内容的不同修改）
- 做：两分支改同一行制造冲突；手动解决并合并
- 链接：[Git 教程目录（廖雪峰）](https://liaoxuefeng.com/books/git/index.html)

- 做：按 UE 文档检查 VS 组件；创建一个 C++ 控制台项目并断点单步
- 链接：

### Day 4（UE C++ Quick Start：创建工程）
- 学：UE C++ 工程结构、创建类、编译运行
- 做：按 Quick Start 完成“新建工程 + 新建 Actor + 运行”
- 链接：[UE C++ Quick Start（官方）](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/unreal-engine-cpp-quick-start)

### Day 5（日志：UE_LOG 与输出窗口）
- 学：UE 日志类别、级别、查看 Output Log
- 做：在 BeginPlay/Tick 打印 UE_LOG；练在输出里筛选关键日志
- 链接：[虚幻中的日志记录（官方）](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/logging-in-unreal-engine)

### Day 7（轻量：代码风格/目录整理）

- 做：整理 `Source/Public/Private`；统一命名；补 `.gitignore`

#### 每周验收点
- 能从 0 创建 UE C++ 工程并运行 Actor 示例
- Git：能独立处理一次冲突并合并
- 日志：能用 UE_LOG + Output Log 复盘一次运行链路

#### 推荐练习题
- 写 `AHelloActor`：BeginPlay 打一次；Tick 每秒打一次计数
- Git：模拟“两个分支改同一文件”并保留双方改动合并

## Week 2｜C++ 基础 + 调试能力（系统岗硬技能）
### 本周目标
- 形成“崩溃定位 SOP”：复现 → 调试器 → 调用栈 → 变量 → 根因 → 修复
- 建立 RAII/所有权直觉（为 UE 生命周期/GC 打底）

### Day 1（C++ 文档入口 + 基本编译直觉）
- 学：MSVC/标准库文档入口；编译/链接概念（知道去哪查）
- 做：浏览 C++ 文档目录；记录 5 个你常用的查询入口
- 链接：[Microsoft C++ 文档入口](https://learn.microsoft.com/zh-cn/cpp/?view=msvc-170)

### Day 2（调试器三件套：断点/单步/调用栈）
- 学：Step Into/Over/Out；Call Stack；Watch
- 做：写 3 层函数调用；从崩溃点回溯调用栈
- 链接：[VS 调试 C++ 入门](https://learn.microsoft.com/zh-cn/visualstudio/debugger/getting-started-with-the-debugger-cpp?view=visualstudio)

### Day 3（快速上手：断点 + 变量检查）
- 学：启动调试器、逐步执行、检查变量
- 做：练条件断点（例如 i==100 才停）；练 Watch/Autos
- 链接：[调试 C++ 代码：断点与变量检查（Microsoft）](https://learn.microsoft.com/zh-cn/visualstudio/debugger/quickstart-debug-with-cplusplus?view=visualstudio)

### Day 4（RAII：用析构“自动收尾”）
- 学：资源获取即初始化；析构保证释放
- 做：写 `ScopeTimer`：构造记录时间，析构打印耗时
- 链接：[C++ Core Guidelines（中文译本）](https://github.com/lynnboy/CppCoreGuidelines-zh-CN)

### Day 5（const / 引用 / 指针：把接口写得难用错）
- 学：`const&` 传参；指针语义；空指针风险
- 做：把一组函数参数从按值改为 `const&`；用调试器观察拷贝/地址
- 链接：[cppreference 中文（语言与标准库索引）](https://cppreference.cn/w/)

### Day 6（复盘：写你的“崩溃定位 SOP”）
- 学：工程化表达（现象→证据→根因→修复）
- 做：写 `docs/debug-sop.md`：最少包含调用栈截图/日志片段/修复 commit
- 链接：[VS 调试 C++ 入门（复查流程）](https://learn.microsoft.com/zh-cn/visualstudio/debugger/getting-started-with-the-debugger-cpp?view=visualstudio)

### Day 7（轻量：做一个“Bug 关卡”并独立修复）
- 学：常见 bug 类型：越界/空指针/未初始化
- 做：写 3 个最小 bug（各 10 行以内）；按 SOP 修复
- 链接：[调试 C++：快速入门（Microsoft）](https://learn.microsoft.com/zh-cn/visualstudio/debugger/quickstart-debug-with-cplusplus?view=visualstudio)

#### 每周验收点
- 能用 1–2 分钟讲清一次崩溃：发生位置/调用栈/根因/修复
- 你写的 `ScopeTimer` 能在任意函数块里复用并输出耗时

#### 推荐练习题
- `UniqueBuffer`：禁止拷贝、允许移动（理解所有权转移）
- 条件断点：只在某个输入满足时停（提升定位效率）

## Week 3｜容器/算法 + 小模块接口设计（为读 UE 代码准备）
### 本周目标
- 熟练 `vector/string/map/unordered_map`
- 写出可复用的小模块（后续 UI/Subsystem 会复用）

### Day 1（vector/string：容量与越界）
- 学：reserve、push_back、迭代、越界风险
- 做：写一个动态列表；加入边界检查；制造一次越界并修复
- 链接：[cppreference 中文（容器/算法）](https://cppreference.cn/w/)

### Day 2（map vs unordered_map：查找结构）
- 学：有序/无序映射的差异与使用场景
- 做：写 `Registry<ID,Obj>`；提供 `Find/Insert/Remove`
- 链接：[cppreference 中文（map/unordered_map）](https://cppreference.cn/w/)

### Day 3（事件系统雏形：观察者/订阅发布）
- 学：观察者模式；订阅/取消订阅；避免悬空回调
- 做：实现 `EventBus`：Subscribe/Unsubscribe/Publish
- 链接：[设计模式目录（Refactoring.Guru 中文）](https://refactoringguru.cn/design-patterns/catalog)

### Day 4（调试容器 bug：定位越界/无效迭代器）
- 学：容器 bug 的典型调用栈表现
- 做：故意写一个“迭代时 erase”的 bug；用调试器定位并修复
- 链接：[VS 调试 C++ 入门](https://learn.microsoft.com/zh-cn/visualstudio/debugger/getting-started-with-the-debugger-cpp?view=visualstudio)

### Day 5（接口设计：最小暴露 + 所有权明确）
- 学：API 设计的“难用错”原则；返回值/错误处理
- 做：给 EventBus 增加“订阅句柄”（RAII 自动取消订阅）
- 链接：[C++ Core Guidelines（中文译本）](https://github.com/lynnboy/CppCoreGuidelines-zh-CN)

### Day 6（复盘：写 README + 用例）
- 学：文档化模块（系统向必备）
- 做：为 EventBus 写 `README.md`：用例、注意事项、复杂度
- 链接：[Refactoring.Guru（观察者相关说明入口）](https://refactoringguru.cn/design-patterns/catalog)

### Day 7（轻量：代码清理 + 风格一致）
- 学：统一命名、减少重复、补断言
- 做：清理重复逻辑；对外 API 加断言/校验；整理目录
- 链接：[Epic C++ 代码规范（先对齐命名）](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/epic-cplusplus-coding-standard-for-unreal-engine)

#### 每周验收点
- EventBus 可用：能订阅/取消/广播；无明显悬空回调问题
- 能独立定位并修复一次容器相关 bug（越界/无效迭代器）

#### 推荐练习题
- EventBus：支持“一次性订阅”（触发后自动注销）
- 写 `Result<T>`：成功/失败（为后续系统返回值规范化做铺垫）
